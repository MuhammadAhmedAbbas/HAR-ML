TECHNICAL DETAILS FOR DEVELOPERS
═══════════════════════════════════════════════════════════════════════════════

FEATURE EXTRACTION
──────────────────

OLD CODE (predator.py):
  - Selected 12 specific landmarks: [11, 12, 13, 14, 15, 16, 23, 24, 25, 26, 27, 28]
  - Extracted only (x, y) — 24 features total
  - Custom normalization using torso length

NEW CODE (predictor_improved.py):
  - ALL 33 MediaPipe landmarks
  - All 4 values: (x, y, z, visibility)
  - 132 features total
  - StandardScaler normalization
  - Consistent with preprocessing and training


TEMPORAL MODELING
─────────────────

Data Structure: (Batch, 30 frames, 132 features)

Sliding Window Creation:
  - Extract every 2nd frame from video (FRAME_SKIP=2)
  - Create 30-frame windows
  - Stride of 15 frames (50% overlap)
  - Result: More training samples, temporal correlation

LSTM Architecture:
  Input → LSTM(128) → Dropout(0.4) → LSTM(64) → Dropout(0.4) 
          → Dense(64, relu) → Dropout(0.4) → Dense(6, softmax)

This captures:
  1. Sequential patterns in body movement (LSTM)
  2. Complex feature interactions (Dense layers)
  3. Prevents overfitting (Dropout + BatchNorm)


INFERENCE PIPELINE
──────────────────

Per Frame:
  1. Detect pose landmarks → 132 features
  2. Apply StandardScaler normalization
  3. Add to 30-frame buffer

When Buffer Full (30 frames):
  1. Pass through LSTM → Get probability for each class
  2. Apply confidence threshold (50%)
     - If max prob ≤ 50% → "Uncertain"
     - Else → Predicted class
  3. Add to prediction history (last 5 predictions)

Smoothing:
  1. Collect last 5 predictions
  2. Find most common class (majority vote)
  3. Average confidence across 5 predictions
  4. Return smoothed (class, confidence)

This reduces:
  - Frame-to-frame flickering
  - False positives from noise
  - Improves perceived stability

═══════════════════════════════════════════════════════════════════════════════
TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Q: Getting FileNotFoundError when running GUI
A: Run preprocessing and training first:
   python setup_and_train.py

Q: Out of memory during training
A: Reduce batch size in train_improved.py:
   BATCH_SIZE = 8  # or 4

Q: Low accuracy despite retraining
A: Check logs:
   - logs/confusion_matrix.png (which classes are confused?)
   - logs/lstm_improved_evaluation.txt (per-class accuracy)
   - Try increasing EPOCHS to 150

Q: "Cannot find video" error
A: Ensure video format is .avi, .mp4, .mov, or .mkv
   Convert if needed using FFmpeg

Q: Predictions are "Uncertain"
A: This is normal if confidence is < 50%
   Lower threshold in predictor_improved.py:
   CONFIDENCE_THRESHOLD = 0.35  # or lower
