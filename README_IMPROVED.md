# 🏃 Human Activity Recognition (HAR) — Desktop Application

**Accurate activity recognition from video using MediaPipe + LSTM**

## ✨ Features

This application recognizes **6 human activities** from uploaded video files:

| Activity | Label |
|----------|-------|
| 🚶 Walking | 0 |
| 💨 Running | 1 |
| 🏃 Jogging | 2 |
| 🥊 Boxing | 3 |
| 👋 Hand Waving | 4 |
| 👏 Hand Clapping | 5 |

### Key Improvements

✅ **Consistent Feature Extraction** — Uses all 33 MediaPipe landmarks × 4 features = 132 features (no inconsistency)  
✅ **Proper Temporal Learning** — 30-frame sequences with LSTM for better temporal understanding  
✅ **Confidence Thresholding** — Low-confidence predictions labeled as "Uncertain"  
✅ **Prediction Smoothing** — Majority voting over multiple frames reduces flickering  
✅ **Class Balancing** — Data augmentation for imbalanced classes  
✅ **Comprehensive Evaluation** — Confusion matrix, classification reports, training visualizations  
✅ **Video Upload Only** — Clean, focused UI without webcam complications

## 🚀 Quick Start

### Step 1: Preprocess Dataset

This extracts MediaPipe keypoints from all KTH videos and creates training sequences.

```bash
python preprocessing/preprocess_improved.py
```

**Output:**
- `dataset/X_train.npy` (training sequences)
- `dataset/X_test.npy` (test sequences)  
- `dataset/y_train.npy` (training labels)
- `dataset/y_test.npy` (test labels)
- `models/scaler_improved.pkl` (feature scaler)

**Time:** ~15-30 minutes (first run)

### Step 2: Train Model

Trains an LSTM neural network with the preprocessed data.

```bash
python training/train_improved.py
```

**Output:**
- `models/har_lstm_improved.h5` (trained model)
- `models/label_encoder_improved.pkl` (label encoding)
- `logs/confusion_matrix.png` (confusion matrix visualization)
- `logs/training_history.png` (loss/accuracy graphs)
- `logs/lstm_improved_evaluation.txt` (metrics report)

**Time:** ~10-20 minutes (on CPU), ~2-5 minutes (on GPU)

### Step 3: Run GUI Application

Once trained, launch the desktop application:

```bash
python gui/app_improved.py
```

## 🎮 Using the Application

### Main Window

1. **📁 Upload Video** — Select a video file (.avi, .mp4, .mov, .mkv)
2. **▶ Start Detection** — Begin real-time activity prediction on the video
3. **⏹ Stop** — Pause the detection
4. **❌ Exit** — Close the application

### Display

- **Left Panel** — Video playback with skeleton overlay
- **Right Panel** — 
  - **Activity Label** — Detected activity in large text
  - **Confidence %** — Prediction confidence level
  - **Prediction Log** — History of recent predictions

### Confidence Thresholding

- ✅ If confidence > 50% → Shows activity name + confidence
- ❌ If confidence ≤ 50% → Shows "Uncertain"
- 🔄 Buffering → Shows "Buffering..." until 30 frames collected

### Prediction Smoothing

To reduce flickering from frame-to-frame noise, predictions use a **majority voting window** over the last 5 predictions. This produces more stable, consistent results.

## 🔧 Technical Details

### Feature Extraction

**Consistent across all pipelines:**
- **33 MediaPipe landmarks** per frame
- **4 values per landmark**: x, y, z coordinates + visibility score
- **Total: 132 features** per frame

### Training Configuration

```
Sequence Length:     30 frames
Sliding Stride:      15 frames
Frame Skip:          2 (sample every 2nd frame from video)
Total Train/Test:    80/20 split
Batch Size:          16
Epochs:              100 (with early stopping if no improvement)
```

### Model Architecture

```
Input: (batch, 30, 132)
  ↓ LSTM(128 units, return_sequences=True)
  ↓ BatchNormalization
  ↓ Dropout(0.4)
  ↓ LSTM(64 units)
  ↓ BatchNormalization
  ↓ Dropout(0.4)
  ↓ Dense(64, relu)
  ↓ Dense(6, softmax)  ← 6 activity classes
Output: [0,1]  (probability per class)
```

### Inference Pipeline

1. **Frame by frame extraction** → 132 features
2. **Feature normalization** → Apply feature scaler
3. **Buffer accumulation** → Collect 30 frames
4. **LSTM prediction** → Get probability for each class
5. **Confidence threshold** → If prob ≤ 50.0% → "Uncertain"
6. **Temporal smoothing** → Majority vote over last 5 predictions

## 🐛 Troubleshooting

### Model Not Found

```
FileNotFoundError: Model not found!
```

**Solution:** Run preprocessing then training:
```bash
python preprocessing/preprocess_improved.py
python training/train_improved.py
```

### Out of Memory During Training

Reduce batch size in `train_improved.py`:
```python
BATCH_SIZE = 8  # or 4 for very low memory
```

### Webcam Still Showing (Old GUI)

Only use the **new GUI**:
```bash
python gui/app_improved.py
```

Don't use the old GUI:
```bash
# DON'T use this:
python gui/app.py
```

### Low Accuracy

1. **Check class balance:**
   ```bash
   tail -20 logs/lstm_improved_evaluation.txt
   ```

2. **Review confusion matrix:**
   - Look at `logs/confusion_matrix.png`
   - Which classes are confused?

3. **Retrain with more epochs:**
   - Increase `EPOCHS = 150` in `train_improved.py`
   - Increase `EARLY_STOP_PATIENCE = 20`

### Videos Not Processing

Supported formats: `.avi, .mp4, .mov, .mkv`

If still not working:
- Try converting video to .avi or .mp4 with FFmpeg
- Ensure video has readable frame rate
